## 2026-06-25 — Юнит-экономика после гранта + Happ/Hiro + Daniil выбит жёстким БС

Длинный заход (продолжение 06-24): экономика → решения по костам → конкурент → живой саппорт.
- **Юнит-экономика (анализ: биллинг YC + БД + `/proc/net/dev`):** грант Яндекса кончился 06-19, владелец пополнил баланс (суспенд снят, ~недели). Структурно сервис **убыточен ~−2300₽/мес** (доход ~3500₽: 12 платящих, июнь 3540₽ ручных оплат vs инфра ~5800₽: eu1 936 + Timeweb 458 + YC ~4400 нетто). Грант давал ~4400₽/мес скрытой субсидии (был +2300 → стал −2300). Брейк-ивен ≈ **30–50 платящих**. Корень — метрированный Яндекс-трафик релея (yc ~1330 + yc2 ~900 ГБ/мес; yc2 **НЕ простаивает** — телеметрия его не опрашивала, проверено `/proc/net/dev`). Память `project_vpn_server_costs_cliff`.
- **Решения владельца:** трафик НЕ режем (свобода юзерам), Timeweb оставляем, тарифы давить тяжело, **yc2 → на сворачивание, пересмотр конец июля** (экономит ~600–900₽ VM+диск+IP; трафик переедет на yc). В ROADMAP.
- **Happ удалён из РФ App Store** (новость) — бьёт онбординг новых iOS; Android ок; durable = иностранный Apple ID / альт-клиент (тест на устройстве). Память `project_vpn_client_happ_reference`.
- **Конкурент Hiro VPN** (hiro-app.com) → `competitors-analysis.md`: фондируемая компания, своё приложение на всех платформах + роутеры; whitelist-busting не заявляет.
- **Daniil (ПЛАТЯЩИЙ) выбит жёстким БС.** Жалоба «не работает, ни Telegram, ни прочее». Диагноз (серверные факты + живой access-log yc + скрин, не гипотеза): туннель РАБОТАЛ (коннекты в Telegram до 13:35:30 МСК) → 26+ мин ноль от его IP при живом узле; AmneziaWG таймаут 12с; РУ грузится; «не только у него» = **жёсткий whitelist на его сети** (оба транспорта срезаны, наши IP не в белом списке). Дано: смена Wi-Fi→LTE как единственный шанс. **БС впервые режет платящую выручку**, не тест-контакт. Память `project_vpn_hard_whitelist_unbeaten`.
- **Стратегически (честный разбор):** при грей-зоне + потолке дистрибуции реалистичная роль VPN = break-even-опция, не доход; «рост» классическими каналами структурно закрыт; цель — держать дёшево через сарафан, не масштаб. (Личные финансовые детали — вне репо.)

## 2026-06-23 — Whitelist-обход: статика + vk-turn провалены → трек запаркован

Расследование пробоя жёсткого БС (триггер: Аня, Муром, near-constant whitelist; + лист ожидания ~20, «обход разблокирует доступ любому»).
- **Статика (igareck white-list sub, 150 серверов)** — Аня реально подключалась к каждому пингующемуся + проверяла YouTube: **ни один не работает**. «Пинг есть, туннель не несёт» = whitelist режет сам трафик. 2 семейства igareck (CIDR+порт80/WS; REALITY под `x5.ru`/`vk.com`) — оба провалены. Наши ноды тем более (иностранные SNI=microsoft; Россия SNI=cloud.mail.ru). Статический endpoint-bypass для её тира **мёртв** (аренда = повтор провала; yc/yc2 = тот же класс датацентр-IP).
- **vk-turn-proxy PoC** (`cacggghp/vk-turn-proxy` «Good TURN») — туннель WG через whitelisted VK-TURN (WG по loopback, на проводе только «VK-звонок» → WG-блок не помеха; концептуально проходит). Сервер подняли на eu1 изолированно (wg1 + Good TURN :56000), **снесли после теста** (прод не тронут). Клиент не получает TURN-креды: **VK сменил капчу** на `not_robot_captcha`/`session_token`, тулза ждёт старый `captcha_sid` → `missing captcha_sid`, петля. Проверено с **РФ-IP владельца (VPN off) + `-manual-captcha`** — тулза сломана новой капчей VK (не наш сетап/IP/VPN).
- **Доп. барьеры vk-turn для продукта:** iOS-клиент = sideload + $99/год Apple Dev (TestFlight) либо iSH (нетехнарю нереалистично); нужен живой VK-звонок 24/7; гонка вооружений.
- **Итог:** рабочего обхода жёсткого БС сегодня нет. Трек **запаркован**. Revisit-триггер: тулза/форк починят VK-капчу, или появится иной механизм, или спрос оправдает кастом-разработку. Память `project_vpn_hard_whitelist_unbeaten`.

Платные не трогаются. Тест на тест-аккаунте: revoke 3 peers+token+gate+notify ОК, платный выпадает. Мягкий кэп ≤10 мин.
