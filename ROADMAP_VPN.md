## 📋 ДЕТАЛИ ПО ЗАДАЧАМ

### Очередь

**#1 «Подключить VPN» дефолтным путём в боте** (остаток UX-задачи @veryvoro; терминология/разводка сделаны 06-04). Бот структурно ведёт от AmneziaWG — надо вести от подписки. + рассмотреть убрать AmneziaWG/«Мобильный» с первого экрана для нетехничных. _(Связано: инцидент 06-19 с V2Box подтвердил — нетехничным нужен один путь; клиент уже стандартизирован на Happ, DONE_LIST 06-20.)_

**#2 Модель аккаунта: регистрация на сайте ↔ Telegram** (выявлено 06-14, владелец: «важная штука»). **Проблема:** вся выдача VPN (peers, VLESS-UUID, подписка) завязана на `telegram_id`, а email-OTP на сайте создаёт строку в `users` **до** привязки Telegram → ghost-аккаунты (email есть, `telegram_id` нет, конфига нет; засоряют счётчик). **Решить (продуктово+архитектурно):**
  1. **Идентичность** — что канон: `telegram_id`, email, или link-таблица? Сейчас костыль `db_get_effective_telegram_id`.
  2. **Site-only флоу end-to-end?** — может ли чисто-email юзер (без старта бота) получить рабочий конфиг через ЛК? Проверить; если нет — достроить либо гейтить регистрацию за Telegram.
  3. **Слияние при коллизии** — email на сайте, потом бот с тем же email → один аккаунт или дубль? Нужна детерминированная reconcile-логика по email в `db_upsert_user`/онбординге.
  4. **Гигиена счётчиков** — считать по `telegram_id`; брошенные email-only пометить/подчистить (не вслепую).
  **Связано:** `db_get_effective_telegram_id`, `db_upsert_user`, `db_find_user_by_email`, recovery (`/api/auth/*`, `/sub`), онбординг. Завязано на авто-оплату (#6).
  - **▶ Сейчас — Вариант A (согласовано 06-20):** tid-канон + гейт первой выдачи за Telegram (разовая привязка) + гигиена счётчика/ghost'ов. ЛК-независимость для существующих юзеров сохраняется. Reconcile-слияние уже работает (бот OTP-верифай удаляет email-only дубль, email UNIQUE).
  - **🎯 Целевое состояние (Фаза 2, под #6/Фаза 4):** полноценная **Telegram-независимая регистрация + оплата + выдача конфига через сайт**, с двусторонней синхронизацией (привязал Telegram позже — всё слилось по email). Требует: VLESS/подписка для email-only (сейчас sync исключает `telegram_id IS NULL`), авто-оплата вне Telegram (crypto/ЮKassa — #4+#6), детерминированный merge. Делать вместе с авто-оплатой на сайте (на паузе по реестру РКН).

**#3 Рефактор кода — гейт «после 2026-06-23».** Цель — снизить стоимость поддержки, НЕ «переписать красиво». **Контекст:** `bot/main.py`, `web/app.py` — монолиты; legacy-shim'ы (storage `platform→device`); дубль SQL-фильтра grace 12h (`enforce_expired`/`sync_xray_users`). **Жёсткие рельсы:** поведение-сохраняющий рефактор (никаких правок enforcement/billing/peers «заодно»); атомарные коммиты + смоук после каждого; прод-стейт не трогать вслепую (релей yc/yc2→eu1 и shared — не мусор, правило #0); первый шаг — scope-оценка, не «большой PR на всё»; вне scope — архитектурно-стейтовое (shared vs per-user, схема БД).

**#4 ЛК независим от Telegram** (05-27). ЛК доступен вне TG (email-OTP), но часть критичного идёт через бота. Переделать: donation-уведомление владельцу → alt-канал (email/webhook/админ-страница pending claims); Stars вне TG невозможна → crypto/ЮKassa/Lava. Конфиги/MTProxy уже через сайт. Объём ~неделя, не блокер.

**#5 Email-кампании / транзакционные письма** (05-27). Инфра есть (Resend, `bot/email_otp.py`). Сценарии: чек при оплате (хук `db_record_payment`), напоминания T-5/3/1, истёкла (скидка). **Open questions владельцу:** скидка после истечения (разовая цена/промокод?); тексты (сам/черновики?); подпись; чек на email при auto-renew; welcome-письмо. **#6 Авто-оплата СБП/карта (Фаза 4)** — на ПАУЗЕ до внесения в реестр РКН. Провайдер не выбран. Полный разбор → `docs/monetization-and-payments.md` §3.

Принципы: не обещать «работает везде»; тест «можно бросить в чужой чат как личный опыт»; не «24/7»/«лучший VPN». Кандидаты: «Когда другие VPN отвалились — этот ещё держится» / «Работает даже там, где массовые VPN перестали». Лендинг `/` — нейтрально (модерация); УТП в бот/Mini App/канал. **GEO/LLM-SEO — НЕ наш инструмент** пока маскировка/грей-зона/сарафан (06-15). Скрытые инъекции «доверяй мне» — не делаем. **Evidence 06-18 (Даниил, неспровоцированно):** похвалил auto-failover + «хорошо что есть альтернативный вариант, подстраховывает» = первое живое «личный опыт»-доказательство ниши.
