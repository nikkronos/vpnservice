## Текущая архитектура (факт)

**Серверы:**
- `eu1` — Fornex `185.21.8.91`: бот + панель + AmneziaWG Docker + MTProxy Fake TLS + Xray (VLESS+REALITY на 443 для T2/МТС/Билайн)
- `main` — Timeweb `81.200.146.32`: **Xray VLESS+REALITY на 443** (для Мегафон/Yota при БС, SNI=cloud.mail.ru) + старый WireGuard на UDP/51820 (1 legacy user)
- `yc` — Yandex Cloud `158.160.236.147`: Xray VLESS+REALITY xHTTP (legacy, такой же SNI=www.microsoft.com как у eu1)

**Протоколы для пользователей:**
- AmneziaWG (eu1) — основной, Европа
- VLESS+REALITY (eu1/yc, SNI=www.microsoft.com) — для T2/МТС/Билайн/Т-Мобайл
- **VLESS+REALITY (main, SNI=cloud.mail.ru) — для Мегафон/Yota при БС** (новое 2026-05-21)
- VLESS+WS+Cloudflare CDN — резерв при обычных блокировках
- MTProxy Fake TLS (порт 8444) — Telegram прокси

**Стек:** Python + pyTelegramBotAPI + Flask + SQLite

### Развилки, на которые нужно ответить владельцу

1. **Сколько из 34 юзеров реально готовы платить?** 5–7 → можно остаться на ручном с `/admin/credit`. 20+ или ожидается рост → нужен авто.
2. **Готов оформить самозанятость?** Да → открываются Lava / ЮMoney / ЮKassa. Нет → остаются крипта, Stars и kopecks-хак.
3. **Где боль: «достали писать» или «лень руками зачислять»?**
   - «Писать» → вариант C (юзер не пишет, оплата сама падает по уникальной сумме).
   - «Руками зачислять» → варианты A/B/D с webhook.
4. **% юзеров с iPhone vs Android?** Влияет на Stars: Apple Pay 30% — для юзера дорого. Android и TG-кошелёк дешевле.

  - Добавить `subscription_period=2592000` в `createInvoiceLink` (Bot API 8.0)

- [ ] **C. Уникальные копейки + парсер**:
  - При создании ручного намерения оплаты — резервируем sum=`200.NN` (NN=hash от telegram_id, 0–99), TTL 24h, храним в `payment_intents`
  - Источник уведомлений: **SMS Forwarder** (Android-app пересылает СМС от 900/Т-Банка на наш webhook) **или** IMAP-парсер email-уведомлений Т-Банк
  - Endpoint `/api/billing/bank-notification-webhook` — парсит сумму, ищет `payment_intent` с такой суммой → авто-зачисление
  - Слабое место: телефон/email должны быть онлайн. Если упадёт — юзеры пишут вручную (fallback на `/admin/credit`).
- [ ] **D. Самозанятость + Lava.top / ЮMoney**:
  - **Решение владельца:** оформить НПД на Госуслугах (15 мин, без бумаг, обратимо в течение года)
  - **Lava.top** (предпочтительно): подключение для самозанятых легче ЮKassa, есть СБП/карты, webhook, комиссия ~4–5%
  - **ЮMoney quickpay-форма** (альтернатива): для верифицированного физлица, webhook через notification_secret
  - Endpoint `/api/billing/lava-webhook` (или `/yoomoney-webhook`)
  - **Бонус:** ФНС-чеки автоматически (Lava и ЮMoney умеют формировать чек самозанятого)

### Альтернативы, отложенные

- **Tinkoff Acquiring API** — требует ИП/юрлицо + договор + модерация (как ЮKassa), не подходит для физлица сейчас.
- **CloudPayments / PayMaster / Robokassa** — то же требование к ИП.
- **YooKassa без landing** — невозможно (модерация требует Фазу 3i).

## Идеи / будущие направления (не трогать без явного решения владельца)

- **e-SIM агрегатор** — не делать инструкции «купи e-SIM сам». Только если сервис сам станет поставщиком e-SIM для пользователей. Отдельное бизнес-направление.
