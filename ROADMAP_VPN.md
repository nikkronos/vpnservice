## Donation-flow — текущий поток монетизации (✅ 2026-05-27, упрощённый курс)

Владелец 2026-05-27 (вечер): «устал от бесконечной борьбы с провайдерами, делаем простую donation-схему». Стало курсом по умолчанию.

**Как работает (end-to-end):**
1. Юзер в ЛК / в боте видит реквизиты Т-Банк (СБП `+79213032918` или карта `2200 7007 6046 4759`) + кнопку **«✅ Я перевёл деньги, подтверди»**.
2. Жмёт → создаётся `payment_claim` (`status='pending'`) → владельцу в TG приходит сообщение с инлайн-кнопками «✅ Подтвердить +30 дн» / «❌ Отклонить».
3. Владелец проверяет в банке поступление, жмёт «Подтвердить» → бот зачисляет +30 дней через `db_extend_subscription` (стакается с остатком), записывает платёж как `provider='manual'` + `external_id='claim:<id>'` (идемпотентно), применяет реферал-бонус, шлёт юзеру «✅ Подписка продлена до DATE».
4. Если «Отклонить» → юзер получает «Оплата не подтверждена, проверь перевод и нажми ещё раз».
5. **Один pending на юзера** — повторные нажатия только переотправляют уведомление (не плодят дубли).

**Файлы:**
- БД: таблица `payment_claims` + флаги `notif_7d/3d/0d_sent` в `users`.
- Web: `POST /api/billing/claim-payment` (email-auth) → создаёт claim + дёргает TG sendMessage.
- Bot: callback handlers `claim_approve:` / `claim_decline:`, кнопка «💳 Продлить подписку» в главном меню, экран реквизитов с кнопкой «Я перевёл».
- ЛК: пейBlock переключается в pending-режим если `d.pending_claim`.

**Cron-напоминания (✅ 2026-05-27):**
- `scripts/expiry_reminder.py` крутится через cron `0 9 * * *` (12:00 МСК) на Fornex.
- Шлёт юзерам сообщения T-7 / T-3 / T-0 до окончания подписки (точная календарная дата). Идемпотентно через флаги `notif_*_sent`, которые сбрасываются при каждом продлении.
- Grandfather (`expires_at IS NULL`) пропускаются.
- Если юзер заблокировал бота (403) — флаг ставится всё равно (не ретраим).

**Что НЕ требует enforcement.** Текущие 34 grandfather-юзера остаются с `expires_at = NULL` — для них напоминаний не будет, доступ безграничный. Как только первый платит — у него ставится `expires_at = now + 30 дн` и дальше пошла обычная подписка. Это мягкий вход.

**Stars-канал остаётся** (commit `23cf3c5` от утра): две кнопки в ЛК «⭐ Подписка Stars (150⭐/мес, авто)» и «⭐ Оплатить один раз». Авто-зачисление через `successful_payment`-handler, без участия владельца. Не мешают и не дублируют donation-flow — это два параллельных канала.

**`/admin/credit` остаётся** как fallback на случай если бот упадёт или inline-кнопки отвалятся.

- [ ] **УТП и позиционирование** (добавлено 2026-05-22 после фидбэка Дмитрия Царёва):
  - **Проблема:** mass-market VPN (hide-my-name.cloud, avoVPN и др.) предлагают "одна кнопка → подключился" за 185-330₽/мес. Без чёткого УТП нам не объяснить, почему пользователь должен переходить к нам.

  - **Целевая аудитория:** люди, которые уже пробовали hide-my-name/etc и столкнулись с отвалом при БС, whitelist-режиме или новых блокировках РКН. Это не "VPN для бабушки" — это запасной/основной VPN для тех, у кого "не работает".

  ### Selective reset (после broadcast + 24ч)
  Чтобы заставить ленивых перейти, но не отрубить тех, кто уже мигрировал.
  - **Миграция БД**: добавить колонку `users.migrated_at TEXT`.
  - **Хук**: при первом `/start` в новом боте — `UPDATE users SET migrated_at = datetime('now') WHERE telegram_id = ? AND migrated_at IS NULL`.
  - **Команда `/admin/migrate_reset`** (только владельцу):
    - Берёт всех `users` где `migrated_at IS NULL AND active=1 AND telegram_id IS NOT NULL`.
    - Для каждого:
      - Удаляет AmneziaWG peer (`awg set awg0 peer <pubkey> remove` + `amnezia-remove-client.sh` + `amnezia-save-conf.sh` для persist).
      - Удаляет VLESS UUID (через `xray-remove-client.py`).
      - Регенерит `sub_token` → старая subscription-ссылка возвращает пустой ответ.
      - НЕ удаляет запись юзера и его email — он сможет вернуться через новый бот.
    - Возвращает владельцу сводку «Сброшено N конфигов».
  - **MTProxy НЕ трогаем** — это один общий сервер на порту 8444, не привязан к юзерам, открыт для всех (используется кем угодно как Telegram-прокси).
  - **Stars Subscription у непрешедших** — продолжит работать (Telegram авто-списывает раз в месяц), будет накапливать продление в БД, но без AWG-peer / VLESS у юзера всё равно ничего работать не будет. Юзер увидит «оплата прошла, но VPN не подключается» → пойдёт в новый бот → онбординг → новый конфиг → всё запустится. Edge case, но не блокер.

  ### Что юзер увидит со своей стороны
  - Через 24 часа после broadcast: VPN на устройствах отвалится. Открывает Telegram → старый бот `@thisvpnforfriends_bot` не отвечает (он мёртв). Юзер вспоминает broadcast → переходит по ссылке на `@vpnkronos_bot` → онбординг → новый конфиг → подключается.

  - Если уже платил: онбординг → меню → «📲 Получить VPN» → новый конфиг.
  - Если был на триале: онбординг → меню → если триал ещё не истёк, конфиг есть. Если истёк — может оплатить через donation/Stars.

## Идеи / будущие направления (не трогать без явного решения владельца)

- **e-SIM агрегатор** — не делать инструкции «купи e-SIM сам». Только если сервис сам станет поставщиком e-SIM для пользователей. Отдельное бизнес-направление.
