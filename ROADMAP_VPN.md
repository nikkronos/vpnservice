## Текущая архитектура (факт)

**Серверы:**
- `eu1` — Fornex `185.21.8.91`: бот + панель + AmneziaWG Docker + MTProxy Fake TLS + Xray (VLESS+REALITY на 443 для T2/МТС/Билайн)
- `main` — Timeweb `81.200.146.32`: **Xray VLESS+REALITY на 443** (для Мегафон/Yota при БС, SNI=cloud.mail.ru) + старый WireGuard на UDP/51820 (1 legacy user)
- `yc` — Yandex Cloud `158.160.236.147`: Xray VLESS+REALITY xHTTP (legacy, такой же SNI=www.microsoft.com как у eu1)

**Протоколы для пользователей:**
- AmneziaWG (eu1) — основной, Европа
- VLESS+REALITY (eu1/yc, SNI=www.microsoft.com) — для T2/МТС/Билайн/Т-Мобайл
- **VLESS+REALITY (main, SNI=cloud.mail.ru) — для Мегафон/Yota при БС** (новое 2026-05-21)
- VLESS+WS+Cloudflare CDN — резерв при обычных блокировках
- MTProxy Fake TLS (порт 8444) — Telegram прокси

**Стек:** Python + pyTelegramBotAPI + Flask + SQLite

## Открытые задачи

### 🔴 Приоритет 1 — Нужно для коммерциализации

- [ ] **УТП и позиционирование** (добавлено 2026-05-22 после фидбэка Дмитрия Царёва):
  - **Проблема:** mass-market VPN (hide-my-name.cloud, avoVPN и др.) предлагают "одна кнопка → подключился" за 185-330₽/мес. Без чёткого УТП нам не объяснить, почему пользователь должен переходить к нам.
  - **Наша реальная ниша (не "для всех"):** "VPN, который работает когда другие отваливаются" — обходит белые списки РКН (Yota подтверждено, Мегафон в процессе), использует современные antiDPI-протоколы (REALITY/AmneziaWG/XHTTP), не зависит от продавленных классических протоколов.
  - **Целевая аудитория:** люди, которые уже пробовали hide-my-name/etc и столкнулись с отвалом при БС, whitelist-режиме или новых блокировках РКН. Это не "VPN для бабушки" — это запасной/основной VPN для тех, у кого "не работает".
  - **Что сделать:**
    1. Сформулировать УТП в одно предложение (для бота, recovery-сайта, любых будущих лендингов).
    2. Адаптировать тексты в боте под это позиционирование (вместо "Получить VPN" — "Подключиться", выделять "работает когда другие не работают").
    3. Добавить в `docs/commercialization-prelaunch-plan-2026-04.md` или новый документ "позиционирование+messaging".
  - **Связано:** `docs/competitors-analysis.md` (обновлён 2026-05-22 с разбором hide-my-name).

  - **Почему это важно сейчас:** конкурент VPN Пельмень (Instagram-реклама) сделал это ключевой фичей. SPACE Connect тоже заявляет. У нас же есть **готовая серверная инфраструктура** (`unified` профиль), но она не вынесена в UI — то есть мы недопродаём свою же возможность.

    1. Проверить актуальность списка заблокированных доменов в `unified_ss_dst` (есть ли там YouTube, Instagram, ChatGPT, Twitter и т.д. — основные что юзеры реально хотят обойти).
    2. Добавить в бота кнопку выбора профиля при выдаче AmneziaWG: «Обычный (всё через VPN)» / «Универсальный (только заблокированные через VPN)». Логика в `bot/wireguard_peers.py` (`profile_type`) уже есть.
    3. Документировать в инструкциях бота преимущество для юзера: «Госуслуги, банки, Яндекс Карты работают как обычно — без проблем с геолокацией».
  - **Стратегия:** это может стать частью УТП — «VPN который не мешает российским сервисам». Понятная для нетехнического юзера ценность.

## Идеи / будущие направления (не трогать без явного решения владельца)

- **e-SIM агрегатор** — не делать инструкции «купи e-SIM сам». Только если сервис сам станет поставщиком e-SIM для пользователей. Отдельное бизнес-направление.
- **Relay через Yandex CDN** — единственный перспективный путь при whitelist. Сложно, риск нарушения ToS Яндекса. Требует разведки (какие IP реально в whitelist).
