# SESSION_SUMMARY_2026-02-09 — Проект VPN

## Контекст работы

- Пользователь хочет создать собственный VPN/Proxy сервис:
  - приоритет — хорошая скорость, адекватный пинг, простое восстановление при сбоях;
  - целевая аудитория — сам владелец, друзья и коллеги, затем возможная коммерциализация;
  - UX-ориентир — Telegram-боты по типу `@hitvpnbot`, `@barinvpn_bot`, `@avoVPN_bot`.
- Целевые платформы пользователей: Windows и iOS (подключение через стандартные клиенты WireGuard).
- Проект должен коммититься в отдельный Git-репозиторий, по аналогии с проектом «Копия иксуюемся» (`xxx/`).

## Важные изменения и решения

- Выбран базовый технический стек:
  - WireGuard как основной VPN-протокол для MVP (высокая скорость, низкий overhead, удобные клиенты для Windows/iOS).
  - Telegram-бот как основной интерфейс для пользователей (выдача конфигов/QR, статус доступа, регенерация ключей).
- Принято решение вести проект VPN в отдельном Git-репозитории:
  - локальная папка `VPN/` внутри `Cursor_test`;
  - свой `.git` и remote на GitHub (название репозитория будет уточнено).
- Определены основные этапы развития:
  - Этап 0 — подготовка (структура, репозиторий, выбор VPS-провайдера).
  - Этап 1 — MVP (одна WireGuard-нода, конфиг для владельца, тесты на Windows/iOS).
  - Этап 2 — Telegram-бот self-service для нескольких пользователей.
  - Этап 3 — масштабирование и надёжность (несколько нод, health-check, инструкции по аварийному восстановлению).
  - Этап 4 — коммерциализация (тарифы, оплаты, аналитика).
