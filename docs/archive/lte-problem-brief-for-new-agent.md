## Диагностика, которую проводили

1. **tcpdump на eu1** во время попытки подключения с LTE:  
   `tcpdump -n -i eth0 'tcp and dst host 185.21.8.91 and (dst port 443 or 4443)'`  
   → **0 packets captured** — пакеты до сервера не доходят вообще.

2. **Браузер на телефоне** (LTE, VPN выкл.):
   - `http://185.21.8.91` → тайм-аут
   - `https://sub.vpnnkrns.ru` → тайм-аут (Cloudflare тоже не открывается)

3. **journalctl -u xray** во время тестов с LTE:  
   → Нет ни одной строки `accepted` — сервер ничего не получает.

4. **curl с ПК** (не LTE):  
   `curl http://185.21.8.91/vpn` → Xray отвечает (400 Bad Request, ожидаемо для WS без Upgrade).

**Вывод из диагностики:** блокировка происходит на уровне мобильной сети — пакеты физически не доходят до любого зарубежного IP, включая Cloudflare CDN.

## Ключевое ограничение (что нужно учитывать)

Это **не** DPI/обфускация — протокол не имеет значения. Оператор просто **не пропускает пакеты** до IP-адресов, не входящих в whitelist. Соответственно:

- Смена порта (80, 443, 8443, любой) — не поможет.
- Смена протокола (UDP→TCP, REALITY, gRPC, WebSocket, XHTTP) — не поможет.
- CDN-прослойка (Cloudflare, другие) — не поможет, если IP CDN тоже не в whitelist.
- Смена шифрования / обфускации — не поможет.

Поможет только маршрут через IP, который **уже разрешён** в whitelist операторов.

## Что не пробовали (потенциальные идеи)

- **e-SIM иностранного оператора** (например, европейского) — выход за рамки российских ограничений.
- **Выяснить, что конкретно в whitelist** — возможно, там есть что-то, через что можно туннелировать (корпоративные VPN-шлюзы, специфичные CDN, государственные ресурсы с туннелированием).
- **ICMP / DNS tunneling** — экзотика; DNS может быть в whitelist (нужна проверка).
- **Steganography-based** решения поверх разрешённых протоколов.
- **Спутниковый интернет** (Starlink и аналоги) — обходит мобильных операторов полностью.
