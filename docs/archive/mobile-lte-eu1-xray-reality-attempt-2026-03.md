# Попытка обхода блокировки мобильного интернета (LTE): Xray VLESS+REALITY на eu1

**Дата:** март 2026  
**Статус:** с **Wi‑Fi** туннель работает; с **мобильной сети (LTE)** до VPS **eu1 (Fornex)** входящий TCP **не доходит** — обход на eu1 для LTE **не достигнут**.

---

## 1. Исходная проблема

- **AmneziaWG** и **Telegram-прокси** стабильно работают по **Wi‑Fi** (разные сети).
- По **мобильному интернету** (LTE, разные операторы и телефоны): тайм-ауты AmneziaWG, прокси в «подключение» / «проверка».
- **Россия (main)** и **Европа (eu1)** по сотовой сети — симптомы схожи для VPN/прокси до зарубежного IP.
- Цель: **полный VPN** с телефона; готовность к параллельному стеку (TCP, REALITY).

## 3. Что сделано на сервере eu1 (Fornex, 185.21.8.91)

1. **Бэкап** AmneziaWG/Docker в `/root/vpn-backups/…-pre-xray-reality` (inspect, `awg0.conf`, копия `amnezia-backup-*`).
2. Установка **Xray-core** (v25.3.6), ключи **`xray x25519`**, UUID клиента.
3. **VLESS + flow `xtls-rprx-vision` + REALITY**, маскировка `dest` / SNI: `www.microsoft.com:443`.
4. Порт **443/tcp** (свободен; **8443** на хосте был занят `docker-proxy` — не использовали для Xray).
5. Второй inbound на **4443/tcp** (дубликат того же REALITY) — для проверки «не режут ли только 443».
6. **systemd** `xray.service`, `ufw allow 443/tcp`, `ufw allow 4443/tcp`.
7. Клиенты: **Streisand** (iOS), ссылка `vless://…@185.21.8.91:443` (и вариант с `:4443`).
8. **Панель Fornex → Firewall:** входящий фаервол **выключен** — блок не на стороне хостера по этой панели.

**Проверка ключа:** `privateKey` в `config.json` совпадает с бэкапом (`CGP0RDQ` — ноль, не буква O).

## 4. Что пробовали для LTE (без успеха)

| Действие | Результат |
|----------|-----------|
| **Fragment** в Streisand | На фоне отсутствия TCP до сервера эффекта не дало; для Wi‑Fi можно держать выкл. |
| Второй порт **4443** | Поведение как у 443: с LTE туннель не устанавливается. |
| **`journalctl -u xray`** за окно теста с LTE | **Нет** новых строк `accepted` — соединения до Xray с телефона по LTE не фиксируются. |
| Safari на **https://185.21.8.91** без VPN, только LTE | Долгая загрузка / тайм-аут — HTTPS до IP VPS с мобильной сети не отрабатывает. |
| **`tcpdump` на eth0** `dst host 185.21.8.91 and (dst port 443 or 4443)` во время попытки с LTE | **0 packets captured** — входящих пакетов на 443/4443 с LTE не видно. |
| Общий **`tcpdump` `port 443 or 4443`** | Пакеты есть, но это преимущественно **исходящий** трафик с **Docker/Amnezia** (`172.29.172.2` → интернет), а не вход клиента на Xray. |

## 5. Вывод (гипотеза)

С высокой вероятностью **мобильный оператор или транзит до Fornex** **не пропускает** (или «чёрная дыра») **TCP до IP `185.21.8.91`** на портах **443** и **4443** (и, вероятно, шире — по IP/ASN).

- Это **не** ошибка конфигурации REALITY на eu1 для сценария «клиент дошёл до сервера»: с **Wi‑Fi** тот же конфиг **работает**.
- Это **не** внешний firewall Fornex (в панели выключен) и **не** отсутствие слушателя: `ss` показывает **xray** на `*:443` и `*:4443`.

## 6. Что делать дальше (когда вернёшься к задаче)

### Приоритет A — другой «первый хоп» с LTE

1. **Поднять такой же Xray VLESS+REALITY на main (Timeweb, 81.200.146.32)**  
   - Выбрать **свободный TCP-порт** (если 443 занят вебом/другим — например 8443, 4443, 2053).  
   - Не сломать существующие сервисы (бот, панель, WireGuard **51820/udp**).  
   - Новая `vless://` с IP **Timeweb**, добавить в бот (сейчас одна переменная `VLESS_REALITY_SHARE_URL` — либо временно заменить ссылку, либо доработать бот: две ссылки «EU / RU mobile»).  
   - Трафик пойдёт с **российского** IP; для части заблокированных сервисов может отличаться от eu1, зато **мобильный доступ** часто восстанавливается именно так.

2. **Другой VPS / другой провайдер / другой ASN** (не Fornex или другая подсеть), пока его не занесли в те же списки — повторить тот же стек REALITY.

### Приоритет B — продукт / безопасность (MVP)

- **Ротация ключей** и **отдельные UUID на пользователя** вместо одной общей ссылки.  
- Две ноды в боте: eu1 (Wi‑Fi / ПК) + «mobile» (RU или другой IP).

### Диагностика (повторить при новом IP)

```bash
# На VPS, только LTE на телефоне, VPN выкл.
sudo tcpdump -n -i eth0 'tcp and dst host ВАШ_IP and (dst port 443 or dst port 4443)' -c 20
```

Если появляются SYN — до сервера доходят, копать REALITY/клиент. Если **0** — снова блок до IP.
