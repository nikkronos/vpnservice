# Спецификация: Европа через Remnawave, бот выдаёт ссылку подписки

**Цель:** Для сервера «Европа» использовать **Remnawave** (Xray VLESS/Reality) вместо AmneziaWG. Пользователь выбирает Европу, нажимает /get_config и получает **ссылку подписки** (subscription URL) от бота. Одна рекомендация клиента на платформу (MVP).

**Дата:** 2026-02-23

---

## 1. Контекст

- **Проблема:** AmneziaWG на eu1 — конфиг от бота на телефоне даёт handshake, но трафик не идёт; ручной экспорт из приложения у друзей вроде работает. Гипотеза: РКН/сеть или отличия формата конфига. Владелец не хочет циклов «тест–починка», приоритет — другой способ подключения.
- **Решение:** Развернуть **Remnawave** (панель + Xray VLESS/Reality) на сервере; пользователи подключаются по **ссылке подписки**; бот автоматически создаёт пользователя в панели (или выдаёт ссылку) и отправляет ссылку в Telegram.
- **Единая рекомендация:** Один клиент на платформу (ПК, iOS, Android) для MVP; при росте числа пользователей — расширять.

---

## 2. Что такое Remnawave (по инструкции пользователя)

- **Серверная часть:** Установка на VPS скриптом из репозитория `eGamesAPI/remnawave-reverse-proxy`:
  - NGINX + Xray (VLESS Reality), панель Remnawave, страница подписок.
  - Нужен **домен** и 3 поддомена: панель, подписки, нода (SelfSteal).
  - Сертификат: ACME (вариант 2 в скрипте) или Cloudflare.
  - Опционально: отключение IPv6 (unit `disable-ipv6.service`) для единой геолокации.
- **Клиент:** Пользователь получает **subscription URL**, открывает его в поддерживаемом приложении (Clash Verge Rev, Nekoray, Shadowrocket и др.) — конфиг подтягивается автоматически.
- **API:** У Remnawave есть backend с REST API; TypeScript SDK `@remnawave/backend-contract` (типы, без HTTP-клиента). Авторизация: `Authorization: Bearer <API_TOKEN>`. Для автоматической выдачи ссылок боту нужен вызов API: создание пользователя + получение subscription URL.

## 3. Варианты размещения

| Вариант | Сервер | Плюсы | Минусы |
|--------|--------|-------|--------|
| **A. Remnawave на eu1 (Fornex)** | 185.21.8.91 | Один VPS, уже оплачен; MTProto и прочее можно оставить | Нужен домен; сосуществование с текущими сервисами (порты 80/443) |
| **B. Отдельный VPS под Remnawave** | Новый | Чистая установка, без конфликтов | Доп. расходы; два «европейских» входа |

**Выбор:** Развёртывание на **eu1 (Fornex)** — один оплаченный VPS, на другой тратиться пока не планируется. Нужен домен (см. `docs/remnawave-deploy-instruction.md`, раздел «Где взять домен»: свой домен, недорогой .ru, Cloudflare или бесплатный DuckDNS для теста).

## 4. Требования перед установкой

1. **Домен:** Один домен (например, `vpn.example.com`), для которого можно создать A/CNAME записи:
   - `panel.vpn.example.com` → IP сервера (eu1 или новый VPS),
   - `sub.vpn.example.com` → тот же IP,
   - `node.vpn.example.com` → тот же IP (или корень домена).
2. **Сервер:** Ubuntu/Debian, root или sudo; порты 80, 443 свободны для NGINX/Xray (или скрипт сам настроит).
3. **IPv6 (по желанию):** Отключить через `disable-ipv6.service` по инструкции пользователя — для единой геолокации (IPv4 = страна ноды).

---

## 5. Развёртывание Remnawave (пошагово)

См. **`docs/remnawave-deploy-instruction.md`** (создаётся отдельно). Кратко:

1. `sudo apt update && sudo apt upgrade -y`
2. Запуск скрипта:  
   `sudo bash -c "bash <(curl -Ls https://raw.githubusercontent.com/eGamesAPI/remnawave-reverse-proxy/refs/heads/main/install_remnawave.sh)"`  
   Выбрать установку Remnawave (панель + нода на одном сервере), сертификат — ACME (вариант 2).
3. По запросу скрипта ввести домен и поддомены (panel, sub, node).
4. (Опционально) Отключить IPv6 — unit из инструкции пользователя, затем `systemctl enable --now disable-ipv6.service`.
5. Сохранить учётные данные панели и (при наличии) API token для бота.

После установки: создать тестового пользователя в панели, получить subscription URL, проверить подключение с ПК и iOS выбранным клиентом.

## 6. Интеграция бота с Remnawave

### 6.1. Целевое поведение

- При выборе сервера «Европа» и команде **/get_config**:
  - Бот создаёт пользователя в Remnawave Panel (по API) с уникальным именем (например, `tg_<telegram_id>` или `user_<uuid>`).
  - Бот получает **subscription URL** этого пользователя (из ответа API или отдельным запросом).
  - Бот отправляет пользователю сообщение с ссылкой подписки и краткой инструкцией: «Открой ссылку в [рекомендованный клиент].»
- **/regen** для Европы: отозвать/деактивировать старого пользователя в панели (если API позволяет), создать нового, отправить новую subscription URL.

### 6.2. Варианты реализации бота

| Вариант | Описание | Плюсы | Минусы |
|--------|----------|-------|--------|
| **I. API Panel** | Бот вызывает REST API Remnawave: создать пользователя, получить subscription URL. | Полная автоматизация, одна ссылка на пользователя. | Нужна документация/исследование API (backend-contract, репозиторий remnawave/backend). |
| **II. Одна общая ссылка** | В панели создаётся один «общий» пользователь (или по одному на Squad); бот хранит одну subscription URL в env и отправляет её всем при /get_config для Европы. | Простая реализация, не зависит от API создания пользователей. | Нет персонального учёта трафика по пользователям; отзыв доступа только сменой ссылки для всех. |
| **III. Ручное создание + бот шлёт ссылку** | Владелец создаёт пользователей вручную в панели, копирует subscription URL; бот по команде (или по telegram_id) отдаёт заготовленную ссылку из хранилища (файл/БД). | Минимум кода, быстрый старт. | Не самообслуживание: владелец должен вручную создавать пользователя и подставлять ссылку в бота. |

**Рекомендация:** Стремиться к **варианту I**. Если API создания пользователя/получения subscription URL не найдётся или окажется сложным — временно **вариант II** (одна ссылка на Европу), затем переход на вариант I после изучения backend/contract.

### 6.3. Конфигурация бота (env)

- `REMNAWAVE_PANEL_URL` — базовый URL панели (например, `https://panel.vpn.example.com`).
- `REMNAWAVE_API_TOKEN` — токен для REST API (если вариант I).
- Для варианта II: `REMNAWAVE_SUBSCRIPTION_URL` — готовая ссылка подписки, которую бот отправляет при /get_config для Европы.

### 6.4. Логика бота (Europe)

- **Сервер «Европа»:** Не вызывать скрипты AmneziaWG; не выдавать AmneziaWG `.conf`.
- При **/get_config** для Европы: вызвать Remnawave (API или подставить URL из env), отправить пользователю subscription URL + текст инструкции (один клиент на платформу).
- **/instruction** и **/help:** Для Европы — инструкция по Remnawave: «Скачай [клиент], открой ссылку из бота, включи VPN.»
- **/regen** для Европы: при варианте I — пересоздать пользователя в панели, отправить новую ссылку; при варианте II — отправить ту же ссылку или сообщение «ссылка не меняется».

## 7. Единая рекомендация клиента (MVP)

Один клиент на платформу, чтобы не плодить сущности:

| Платформа | Рекомендуемый клиент | Примечание |
|-----------|----------------------|------------|
| **Windows** | Clash Verge Rev или Nekoray | В документации Remnawave указан Clash Verge Rev; Nekoray поддерживает подписки. |
| **macOS** | Clash Verge Rev или Nekoray | Аналогично. |
| **Android** | Nekoray или V2RayNG | Поддержка subscription URL. |
| **iOS** | Shadowrocket (платный) или аналог с подпиской | Указать в инструкции один вариант; при росте MVP — добавить альтернативы. |

Зафиксировать в `docs/client-instructions-remnawave.md` и в текстах бота (`docs/bot-instruction-texts/instruction_remnawave_short.txt`).

## 8. Миграция с AmneziaWG

- Пользователи, у которых уже есть AmneziaWG для Европы: сообщить, что «Европа» переведена на Remnawave; по /get_config они получают ссылку подписки; старый AmneziaWG конфиг больше не выдаётся.
- AmneziaWG на eu1 можно остановить/удалить после успешного запуска Remnawave и проверки друзьями (или оставить как запасной вариант до стабилизации).

---

## 9. Риски и митигация

| Риск | Митигация |
|------|-----------|
| API Remnawave не документирован для создания пользователя | Изучить remnawave/backend, backend-contract; временно вариант II (одна ссылка). |
| Конфликт портов на eu1 (443, 80) | Проверить занятость до установки; при конфликте — отдельный VPS (вариант B). |
| Домен не готов | Отложить установку до появления домена и DNS-записей. |
| Разные клиенты на iOS (платный/бесплатный) | Зафиксировать одну рекомендацию в инструкции; при необходимости сменить позже. |
