# Инструкция: как подключиться к VPN на Android

## Что понадобится

- Файл конфигурации WireGuard (`.conf`) или его текст, выданный администратором или через Telegram‑бота  
- Для сервера **Европа (eu1)** — приложение AmneziaVPN или AmneziaWG (обход блокировок из РФ)

---

## Сервер «Россия» (WireGuard)

### Шаг 1: Установка WireGuard

1. Установи приложение **WireGuard** из Google Play.
2. Открой приложение.

### Шаг 2: Импорт конфига

**Вариант А — из файла**

1. Сохрани присланный файл `.conf` на устройство (например, скачай из сообщения в Telegram в «Загрузки»).
2. В WireGuard нажми **«+»** → **«Импорт из файла или архива»** (или **Import from file**).
3. Выбери сохранённый файл `.conf` (например, из папки Download).

**Вариант Б — из буфера обмена**

1. Скопируй весь текст конфига (от `[Interface]` до конца).
2. В WireGuard нажми **«+»** → **«Импорт из буфера обмена»** (или **Import from clipboard**).

**Вариант В — по QR-коду (если администратор прислал QR)**

1. В WireGuard нажми **«+»** → **«Сканировать QR-код»**.
2. Отсканируй QR-код с конфигом.

### Шаг 3: Подключение

1. В списке туннелей выбери нужный (например, имя из конфига).
2. Включи переключатель — статус сменится на «Подключено», появится значок VPN в панели уведомлений.

### Шаг 4: Типы профилей

- **Обычный VPN** — весь трафик через сервер (YouTube, сайты).
- **VPN + GPT** — тот же сервер, HTTP/HTTPS через Shadowsocks: подходит для ChatGPT и обхода блокировок.

## Сервер «Европа» (AmneziaWG)

Для доступа из РФ к серверу Европа используется протокол AmneziaWG (обфускация). Один конфиг на все устройства.

**Получение конфига:** в Telegram-боте используй **/get_config_android** (вместо /get_config), чтобы получить конфиг без ошибки 1000 на Android. Для обновления ключей — **/regen_android**.

**Если ошибка 1000 всё равно появляется при нажатии «Подключиться»:**
1. В AmneziaVPN удали **все** старые серверы с адресом 185.21.8.91 (Сервер 1, Сервер 2, Сервер 3 и т.д.) — они могли быть добавлены из старого конфига.
2. В боте заново: **/server** → Европа, затем **/get_config_android**.
3. Импортируй **только** этот новый файл .conf (один сервер).
4. Подключись к этому новому серверу. Не смешивай старые и новый конфиги.

### Шаг 1: Установка приложения

1. Установи **AmneziaVPN** или **AmneziaWG** из Google Play: [amnezia.org — Загрузки](https://amnezia.org/en/downloads).
2. Открой приложение.

### Шаг 2: Импорт конфига

1. Сохрани присланный файл `.conf` (AmneziaWG) на устройство (например, из Telegram в «Загрузки»).
2. В AmneziaVPN нажми **«Импорт»** или **«Добавить по файлу»** и выбери сохранённый `.conf`.
3. Либо скопируй текст конфига и вставь через «Импорт из буфера обмена», если приложение это поддерживает.
4. Подключись выбранным профилем.

### Шаг 3: Подключение

1. Выбери созданный туннель AmneziaWG в списке.
2. Включи переключатель — подключение установится.

---

## Отключение

В приложении (WireGuard или AmneziaVPN) выключи переключатель у активного туннеля.

---

## Telegram отдельно (без VPN)

Если нужен только Telegram при блокировках — используй MTProto‑прокси по отдельной ссылке (Настройки → Данные и хранилище → Прокси). Инструкция: `docs/mtproto-client-instructions.md`.

## Проблемы

- **ErrorCode 1000 (ошибка VPN-соединения)** на AmneziaVPN под Android: 1) В боте используй **только** /get_config_android и /regen_android. 2) В приложении **удали все** старые серверы 185.21.8.91 и добавь один новый из присланного .conf. 3) В конфиге должна быть одна запись DNS, например `DNS = 1.1.1.1` (без запятой и второго адреса) — иначе правь вручную.  
  **Если 1000 остаётся:** у AmneziaVPN для Android известна проблема с импортом AmneziaWG-конфигов из сторонних источников (см. [issue #1390](https://github.com/amnezia-vpn/amnezia-client/issues/1390)). Обходы: подключиться с ПК (AmneziaVPN Windows/Mac), экспортировать рабочий конфиг и импортировать его на телефон; попробовать другую сеть (Wi‑Fi / мобильный интернет); написать в поддержку Amnezia (Telegram @amnezia_support_bot или support@amnezia.org). Пока ждёшь — пользоваться VPN с ПК или планшета.
- **Ошибка разрешения DNS**: в конфиге должен быть указан реальный IP сервера в `Endpoint`, а не заглушка.
- **Не подключается**: проверь интернет и доступность сервера (порт 51820/UDP).
- **ChatGPT не открывается**: для Европы используй профиль AmneziaWG; для России — профиль «VPN + GPT», если он выдан.
- **Файл .conf не подхватывается**: убедись, что расширение именно `.conf` и файл не повреждён; попробуй импорт из буфера обмена.
