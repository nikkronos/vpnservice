## 1. Donation-flow — текущий поток монетизации (✅ 2026-05-27, курс по умолчанию)

Владелец 2026-05-27: «устал от борьбы с провайдерами, делаем простую donation-схему».

**End-to-end:**
1. Юзер в ЛК/боте видит реквизиты Т-Банк (СБП `+79213032918` / карта `2200 7007 6046 4759`) + кнопку «✅ Я перевёл деньги, подтверди».
2. Жмёт → `payment_claim` (`pending`) → владельцу в TG инлайн «✅ Подтвердить +30 дн / ❌ Отклонить».
3. Подтверждение → `db_extend_subscription` (+30, стакается) + `db_record_payment` (`provider='manual'`, `external_id='claim:<id>'`, идемпотентно) + реферал-бонус + уведомление юзеру.
4. Отклонение → юзер получает «не подтверждено, проверь и нажми ещё раз».
5. Один pending на юзера (повторные нажатия только переотправляют).

**Файлы:** БД `payment_claims`; web `POST /api/billing/claim-payment`; bot `claim_approve:`/`claim_decline:` + «💳 Продлить подписку»; ЛК pay-block (pending-режим при `d.pending_claim`).

**Напоминания:** см. `scripts/expiry_reminder.py` — ежедневные с 7 дней (с 06-12; было T-7/3/0).

**Параллельные каналы:** Stars (oneshot + Subscription auto-renew, авто-зачисление через `successful_payment`); `/admin/credit` (fallback ручного зачисления).

**Состояние:** Stars-автозачисление закрыто. Для СБП/карты курс не выбран — пока `/admin/credit` вручную. Владелец хочет, чтобы зачислялось само («никто не писал в ЛС после оплаты»).

**Фаза 4 на ПАУЗЕ до реестра РКН** (решение 06-02; уведомление подано 06-02, №100306737).

### Сравнение провайдеров (06-02)

- **Продамус** — под самозанятого лучше: авто-чек НПД встроен, подключение ≈1₽ по промо + грейс без комиссии до 100k/3мес, комиссия 3.5%→2.9%, платёжные ссылки (меньше кода).
- **ЮKassa** — крупнее/надёжнее, бесплатное подключение, НО с 01.2026 +НДС на комиссию (~4.27% эфф.), чек НПД вручную. `docs/yookassa-setup-instruction.md`.
- **Проверить до выбора:** (1) принимают ли категорию (описывать «настройка/сопровождение защищённого сетевого соединения», НЕ «VPN»); (2) у Продамуса 3.5% с НДС или сверху; (3) рекуррент для самозанятого.

### Развилки для владельца
1. Сколько из юзеров реально платят? 5–7 → ручной `/admin/credit` хватит; 20+ → нужен авто.
2. Готов оформить самозанятость? Да → Lava/ЮMoney/ЮKassa.
3. Боль «достали писать» (→C) или «лень руками зачислять» (→A/B/D webhook)?
4. % iPhone vs Android (влияет на Stars: Apple Pay 30%).

### Подзадачи когда выберем курс

- **A. Cryptomus:** мерчант + `/api/billing/create-crypto-invoice` + `/cryptomus-webhook` (HMAC, idempotent) + кнопка в payBlock + ENV `CRYPTOMUS_*`.
- **C. Уникальные копейки + парсер Т-Банк** (закрывает «никто не пишет в ЛС»): intent с суммой `200.NN` (`NN=hash(tid)%100`) → демон ловит уведомление банка → авто-credit. **C1 IMAP-парсер** (рекомендован: не нужен Android 24/7; нужен email Т-Банка + IMAP + пример письма) / **C2 SMS Forwarder Android**. Таблица `payment_intents`, `/api/billing/manual-pay/start`, `/bank-notification-webhook`, `scripts/imap_payment_watcher.py`.
- **D. Самозанятость + Lava.top/ЮMoney:** НПД на Госуслугах → Lava (СБП/карты, webhook, ~4–5%, авто-чек) или ЮMoney quickpay. `/api/billing/lava-webhook`.

**Отложенные:** Tinkoff Acquiring / CloudPayments / PayMaster / Robokassa — требуют ИП/юрлицо.

### Чек НПД — наименование услуги (важно)
НЕ писать «VPN-подписка»/«доступ к интернет-ресурсам» (риск переквалификации + 149-ФЗ). Правильно: «Услуга по настройке и сопровождению защищённого сетевого соединения за 30 дней». Владельцу фиксировать в «Мой налог» при каждом чеке (в код не попадает).
