# Развёртывание MTProxy с Fake TLS (mtg)

Пошаговая инструкция для запуска **своего** Telegram-прокси с маскировкой под HTTPS (Fake TLS). Трафик для DPI выглядит как обычное соединение к легитимному сайту — снижается риск блокировки по сигнатуре. Используется образ **nineseconds/mtg** (не стандартный `telegrammessenger/proxy`).

**Где разворачивать (рекомендация):** сначала попробовать **Timeweb** (81.200.146.32) — сервер в РФ, трафик «абонент → Timeweb → Telegram» часто стабильнее и быстрее; фильтрация на границе к этому каналу не применяется. Если порт 443 на Timeweb занят — отдельный дешёвый VPS в РФ (например, FirstVDS Москва) или другой хостинг.

---

## Требования

- VPS с Linux (Ubuntu/Debian), доступ по SSH (root или sudo).
- **Порт 443 свободен** (для маскировки под HTTPS желательно именно 443).

---

## Шаг 0. Установить Docker (если ещё нет)

На Ubuntu/Debian:

```bash
sudo apt update
sudo apt install -y docker.io
sudo systemctl enable docker
sudo systemctl start docker
docker --version
```

После этого можно переходить к шагу 1.

---

## Шаг 1. Проверить порт 443

На сервере:

```bash
sudo ss -tulpn | grep 443
```

Если вывод пустой — порт свободен. Если занят (nginx, apache и т.п.) — либо освободить 443, либо развернуть прокси на **другом VPS** (на нестандартном порту маскировка менее убедительна).

---

## Шаг 2. Выбрать домен для маскировки

Домен должен быть реальным, с работающим HTTPS (нужен для генерации секрета). Примеры: `1c.ru`, `google.com`, `cloudflare.com`. Лучше популярный и «нейтральный».

Далее в командах используется `1c.ru` — замени на свой при желании.

---

## Шаг 3. Сгенерировать секрет

Один раз выполнить (подставь свой домен вместо `1c.ru`):

```bash
docker run --rm nineseconds/mtg:2 generate-secret --hex 1c.ru
```

В выводе будет строка вида: `eedc591e24110c174ffd773d832dd69b1531632e7275` (префикс `ee` = Fake TLS). **Скопировать и сохранить** — это секрет для ссылки. Не публиковать.

---

## Шаг 4. Запустить контейнер

Подставь **свой секрет** вместо `ВЫВОД_ИЗ_ШАГА_3` и при необходимости свой порт (если не 443):

```bash
docker run -d \
  --name mtproxy-faketls \
  --restart unless-stopped \
  -p 443:443 \
  nineseconds/mtg:2 \
  simple-run -n 1.1.1.1 -i prefer-ipv4 0.0.0.0:443 ВЫВОД_ИЗ_ШАГА_3
```

Пример с секретом (не использовать как есть — сгенерируй свой):

```bash
docker run -d \
  --name mtproxy-faketls \
  --restart unless-stopped \
  -p 443:443 \
  nineseconds/mtg:2 \
  simple-run -n 1.1.1.1 -i prefer-ipv4 0.0.0.0:443 eedc591e24110c174ffd773d832dd69b1531632e7275
```

Проверка:

```bash
docker ps
docker logs mtproxy-faketls
```

Ошибок быть не должно (пустой лог — норма).

---

## Шаг 5. Узнать IP сервера и собрать ссылку

Узнать внешний IP (на самом сервере или из панели хостера):

```bash
curl -s ifconfig.me
```

Ссылка для подключения в Telegram:

```
tg://proxy?server=IP_СЕРВЕРА&port=443&secret=ТВОЙ_СЕКРЕТ
```

Пример (IP и секрет — свои):

```
tg://proxy?server=81.200.146.32&port=443&secret=eedc591e24110c174ffd773d832dd69b1531632e7275
```

Альтернативный формат (открывается в браузере/Telegram):

```
https://t.me/proxy?server=IP_СЕРВЕРА&port=443&secret=ТВОЙ_СЕКРЕТ
```

**Важно:** ссылку не публиковать в открытый доступ; раздавать только себе и доверенным (например, знакомому в Москве для проверки).

---

## Шаг 6. Подключение в Telegram

- **Телефон:** открыть ссылку `tg://proxy?server=...` (в браузере или скопировать в Telegram) → Telegram предложит включить прокси → «Подключить» / «Продолжить».
- **Вручную:** Настройки → Данные и память → Использовать прокси → Добавить прокси → MTProto → ввести Server (IP), Port (443), Secret (секрет из шага 3).

Проверить отправку/получение сообщений, скорость.

---

## Шаг 7. Задокументировать у себя (не в Git)

Сохранить локально (например, в `Main_docs` или в зашифрованном хранилище):

- IP и порт прокси
- Секрет (или только напоминание «хранится в env / таком-то месте»)
- Команды перезапуска (см. ниже)

В репозиторий **не коммитить** саму ссылку и секрет.

---

## Полезные команды

Перезапуск контейнера:

```bash
docker restart mtproxy-faketls
```

Просмотр логов:

```bash
docker logs mtproxy-faketls
```

Остановка и удаление (если переносишь на другой сервер):

```bash
docker stop mtproxy-faketls
docker rm mtproxy-faketls
```

Проверка, что порт слушается:

```bash
sudo ss -tulpn | grep 443
```

---

## Если 443 на Timeweb занят

- Проверить, какой процесс держит 443: `sudo ss -tulpn | grep 443` и при необходимости остановить/перенести сервис (nginx и т.д.) на другой порт.
- Либо развернуть прокси на **отдельном VPS в РФ** (например, FirstVDS Москва) по этой же инструкции — тогда на новом сервере 443 точно свободен.

---

## Если прокси перестанет работать

- Сменить **домен маскировки** (шаг 2–3), пересобрать секрет и перезапустить контейнер с новым секретом.
- Либо перенести контейнер на **другой VPS** (другой провайдер/локация), повторить шаги 3–6 и раздать новую ссылку.

**Автоматизация в проекте VPN:** см. [mtproxy-proxy-rotation.md](mtproxy-proxy-rotation.md) — скрипт ротации, команда бота `/proxy_rotate` (владелец), актуальная ссылка через `/proxy` без ручного правления `env` при каждой смене секрета.
