## 1. Место БД — РЕШЕНО: вариант B (2026-06-02)

**Факт:** бот + веб-панель + `vpn.db` (email + telegram_id юзеров) крутятся на Fornex eu1 = **Германия**. VPN-выходы (AWG/Xray/MTProxy) тоже там, но они ПД не хранят — только маршрутизируют трафик. Вопрос только в базе с ПД.

**242-ФЗ** требует хранить ПД росграждан в базах на территории РФ → БД в Германии этому формально не соответствует (localization gap).

**Решение: остаёмся на Fornex, в форме указываем честно. Инфру НЕ трогаем.** Почему B, а не перенос на RU:
- Приоритет — **не ломать рабочую инфру** > избежать бюрократии в форме.
- Перенос управлялки (бот+БД) на RU имеет реальные риски: **Telegram Bot API из РФ может троттлиться** (главный, непротестированный риск = «бот отвалится»); корреляция отказов (TimeWeb держит REALITY для Мегафон/Yota — подсадка бота туда роняет оба сразу); кросс-граница SSH RU→DE на каждую выдачу конфига; плохой тайминг (UUID-агент в середине задачи).
- localization gap для микро-оператора — **низковероятный** риск (знакомая подавала через нейронку без проблем). И он **не блокирует ЮKassa** технически (ЮKassa проверяет сайт/оферту/контакты, не место БД).

**Перенос на RU (A/C) — отложен, не отменён.** Делать осознанно при росте, пререквизит — **тест Telegram-API из РФ** (нет гипотезы без теста). Бонус: ЛК станет доступнее при БС (TimeWeb whitelisted). Детали в ROADMAP.

> Это архитектурное решение уже принято — остальные поля формы заполняй по разделу 2, поля «трансграничная передача» и «место БД» — по значениям ниже.

## 3. На что НЕ вестись / частые затыки

- **ОГРН нет** — ты самозанятый без ИП. Если поле обязательное и не даёт «отсутствует» — ищи опцию «физическое лицо без статуса ИП».
- **Не указывай** обработку паспортов, биометрии, спецкатегорий — ты их не собираешь (и Политика это прямо отрицает). Чем уже список ПД — тем меньше требований.
- **Согласие на рекламу — отдельная история.** Текущий акцепт на `/recovery` покрывает оферту и Политику (обработку ПД). Но если делаешь **маркетинговые/промо email-рассылки** (задача «Email-кампании» в ROADMAP) — нужно **отдельное согласие на рекламу** (Рина права: «должна быть кнопка»). Сервисные письма (чек, «подписка кончается») рекламой не считаются — для них хватает исполнения договора. Промо («скидка -20%») — реклама, нужна галочка.
