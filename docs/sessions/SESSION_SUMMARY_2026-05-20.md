### 2. Аудит текущего XHTTP relay через Yandex CDN

**Подтверждено:**
- nginx на Fornex корректно роутит: `Upgrade: websocket` → 8080 (WS), всё остальное → 8081 (XHTTP).
- Xray конфиг (`network: xhttp, path: /vpn`) валиден.
- 19.05.2026 в 19:17–19:18 был реальный трафик от двух клиентов (`85.26.163.217`, `87.248.243.135`) с запросами `GET /vpn/<uuid>?x_padding=...` — это валидный XHTTP формат от Xray-клиента. То есть **CDN → Fornex → Xray цепочка проходит**.

**Не подтверждено:**
- С 19 мая 19:18 CDN-relay трафика больше нет. Реальных подключений именно с Yota/Мегафон при активных БС не зафиксировано.
- Не подтверждён сам факт доступности `cdn.vpnnkrns.ru` на Yota/Мегафон при БС.

**Гипотеза причины недоступности (если она есть):**
- Домен `cdn.vpnnkrns.ru` обслуживается через Cloudflare DNS (grey cloud) + CNAME на `*.topology.gslb.yccdn.ru`. При БС могут блокироваться Cloudflare DNS-серверы или сама CNAME-цепочка.
- Yandex CDN IP действительно в whitelist, но клиенту нужно сначала зарезолвить домен — а DNS-провайдер у оператора может не отдавать ответ для нестандартного CNAME при БС.
- Прозрачный прокси Мегафон/Yota для российских доменов (см. `yandex-cloud-reality-setup.md` — фикс REALITY с `.ru` SNI → `www.microsoft.com`).

**Что нужно для финализации диагностики (отдельная задача, не сделано в этой сессии):**
- Тест с реального устройства на Yota без VPN при активных БС:
  - `nslookup cdn.vpnnkrns.ru 8.8.8.8` — резолвится ли домен
  - Попытка открыть `http://cdn.vpnnkrns.ru/vpn` в браузере — что отвечает

### 3. План Yandex API Gateway

Создан документ `docs/yandex-api-gateway-plan.md` — подробный план реализации:

- Архитектура: клиент → `*.apigw.yandexcloud.net` (HTTPS) → API Gateway → HTTP/WS → Fornex
- Преимущество: нативный домен Yandex Cloud, без посредников DNS и без Cloudflare-цепочки
- Шаги: создание API Gateway → проверка доступности на Yota без VPN при БС → адаптация nginx → формирование VLESS-ссылки → обновление кнопки в боте
- Риски: лимиты API Gateway, неподтверждённая поддержка WebSocket в transparent-режиме, latency, ToS YC
- Альтернативы при отказе: Cloud Functions, YC Application Load Balancer + Yandex CDN с тех. поддержкой

**Не реализовано** — ждём момента, когда будет возможность тестировать с устройства на Yota/Мегафон при активных БС.
