## Часть 2 — MTProxy: диагностика и исчерпание

**Симптом:** прокси перестал коннектиться у владельца и других юзеров (подтвердили). Пересборка владельцем утром не помогла.

**Сервер исправен (доказано):** контейнер `mtproxy-faketls` up, 8444 слушается, секрет контейнера == ссылке (`/proxy` отдаёт правильную через override `data/mtproto_proxy_link.txt`; ранее искал не тот путь — `bot/data/` вместо `data/`, ложная тревога рассинхрона снята). Из РФ-дата-центров (main/yc) TCP + fake-TLS проходят. → **проблема не на сервере.**

**Ключевой конфаунд:** все Wi-Fi тесты владельца шли с **включённым VPN** (у него на Wi-Fi VPN всегда вкл, забыл упомянуть). С VPN-on прокси-коннект заворачивается в туннель → тест недействителен. Новое правило в память: тест коннективити прокси — только **VPN OFF**.

**Чистый тест (VPN off) — исчерпание рычагов:**

| Рычаг | Значения | Результат |
|---|---|---|
| Домен fake-TLS | `1c.ru` → `swcdn.apple.com` | без изменений |
| IP | eu1 🇩🇪 `185.21.8.91` / main 🇷🇺 `81.200.146.32` | оба не идут |
| Порт | 8444, 8443, 2053 | ничего |

**Вывод:** ТСПУ режет сам протокол MTProxy по фингерпринту — IP/домен/порт не влияют. Единственный непроверенный рычаг — порт **443** (там fake-TLS сливается с HTTPS), но 443 занят Xray REALITY на всех серверах (Мегафон/Yota при БС) — **не роняли** ради вторичной фичи. mtg сильнее fake-TLS не умеет.

**Действия:**
- **eu1-прокси:** домен сменён `1c.ru`→`swcdn.apple.com` (через `mtproxy-faketls-rotate.sh`), оставлен как канонический. `/proxy` отдаёт его; `data/mtproto_proxy_link.txt` + `env_vars.txt` синхронизированы; в env добавлен `MTPROXY_FAKE_TLS_DOMAIN=swcdn.apple.com` (будущий `/proxy_rotate` не откатит на `1c.ru` — `environment_for_mtproxy_rotate` читает env свежим). Бэкап env: `env_vars.txt.bak.20260605-073224`.
- **main:** тестовые контейнеры (8444/8443/2053) удалены — main снова только Xray/nginx.
- **Прокси-UI заменён на редирект в боте И ЛК** (решение владельца — убрать у юзеров мёртвую ссылку+инструкцию совсем, оставив сообщение + кнопку на VPN; commit `ff24c8c`):
