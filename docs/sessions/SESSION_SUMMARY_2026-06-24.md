## Рефактор #3 — что и почему
Заход начался с продолжения Tier-1 дедупа, но владелец переформулировал цель: важны **токены на чтение + неразбериха агентов в коде**, а не число устранённых дублей. Под эту ось:
- **device-autoname** — чистый byte-identical дубль, свёл в `bot/database.db_device_autoname` (оба call-site: bot `_add_device_and_deliver`, web `api_recovery_device_add`). Локальный функц-тест (ПК 3 / iPhone-iPad 2 / Android 1 / ПК 1) + смоук в ЛК.
- **status-line добито проверкой, а не кодом:** `cmd_status` рендерит богаче (`активна до DATE (N дн осталось)` + иконки, сам считает days_left); support-notify проще (`до DATE`, без счётчика) и его `sub` не содержит days_left. Фолд в `format_subscription_status` изменил/сломал бы вывод → честно оставлены раздельными. Дублей status-line больше нет.
- **Навигация = рычаг 1.** В шапку каждого монолита — карта секций (модульный докстринг); по коду — grep-баннеры `# ═══ §N · NAME` на границах логических групп. `bot/main.py`: §0 модульные хелперы · §1 auth/меню · §2 онбординг · §3 restore/реферал · §4 /start · §5 доставка конфигов · §6 профили/устройства · §7 email · §8 admin · §9 команды · §10 мобильный vless · §11 owner-команды · §12 платежи · §13 support · §14 polling. `web/app.py`: §1-§9 по route-группам. Только комментарии/докстринг → ноль поведенческого риска (py_compile локально+сервер; рестарт не нужен — рантайму безразлично; scp без рестарта для паритета).
- **Глубина — рычаг 2 vs 3.** Проверил выносимость: ~80 хендлеров — вложенные замыкания над `bot`/состоянием, без рычага 3 (`register(bot, deps)`) не выносятся. Чистых «листьев» ~50-80 строк (~2%). Владелец выбрал **закрыть #3**: навигация дала выигрыш по его метрике без риска; рычаг 3 — только если конкретная зона начнёт болеть.
