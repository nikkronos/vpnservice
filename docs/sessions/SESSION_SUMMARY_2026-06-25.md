## Daniil — жёсткий БС (живой саппорт, разбор по фактам)
Платящий (3dev_3m до сентября). Жалоба «не работает, ни Telegram, ни прочее». Диагностика без гипотез:
1. **БД:** доступ активен, подписка/UUID на месте, не гейтнут.
2. **vless_user_traffic:** yc ~147 ГБ lifetime, ~953 МБ за последний интервал — недавно качал тяжело.
3. **Живой access-log yc:** его IP `195.16.60.255` ходил в Telegram (`149.154.x`/`91.108.x`) до **13:35:30 МСК**, потом **26+ мин НОЛЬ** при живом узле (обслуживает других в 14:00).
4. **Скрин клиента:** AmneziaWG → таймаут 12с (UDP/eu1 блок, как 06-23).
5. **От юзера:** РУ-сервисы грузятся; «не только у него»; на Wi-Fi.

**Вывод:** жёсткий whitelist на его сети. Оба транспорта (Яндекс-REALITY yc + немецкий AWG eu1) срезаны на сетевом уровне — наши IP не в белом списке. Конфиги не помогут. Дано: смена Wi-Fi→LTE (единственный шанс — у оператора может быть мягче); иначе ждать снятия (подписка цела, дни не теряются). **Это потолок УТП живьём: продукт упирается в стену БС у реального платящего клиента.** Память `project_vpn_hard_whitelist_unbeaten` (триаж-шаблон по access-log добавлен).

## Стратегия (честный разбор)
- VPN был прибыльным **только из-за гранта**; структурно теперь минус. Это разовая отсрочка, не решение.
- При грей-зоне (нельзя реклама/SEO/блогеры) + потолке дистрибуции (Happ-волны, БС-стена) **рост классическими каналами структурно закрыт** — это не пробел владельца, а свойство ниши.
- Единственный легальный канал — сарафан через момент «у других отвалилось, у тебя работает» (реферал уже есть).
- Согласованная рамка: реалистичная роль VPN = **break-even-опция/актив, не источник дохода**; цель — держать дёшево, не гнаться за масштабом. (Личные финансовые детали владельца — намеренно вне репо.)
