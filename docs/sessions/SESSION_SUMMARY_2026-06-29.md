## Метод ретеста (для воспроизводства)
- **Бинари:** prebuilt релиз форка v1.8.4 (Go, без сборки — локального Go нет). client/server на все платформы. Staged в scratchpad: `client-windows-amd64.exe`, `server-linux-amd64`, `client-linux-amd64`, `captcha_solver.py`. SHA256 зафиксированы в сессии.
- **Команда клиента:** `client-windows-amd64.exe -vk-link "<VK call link>" -manual-captcha -debug -peer <srv>:56000`. Ручная капча отдаётся на `http://localhost:8765` (человек решает в браузере). Авто-режим: `-captcha-solver http://127.0.0.1:8766` (Playwright `scripts/captcha_solver.py`, требует `pip install playwright requests` + `playwright install chromium`).
- **VK call link:** создаётся в VK (Звонки → Создать звонок → Скопировать ссылку); вход анонимный (логин не нужен); ссылка вечная, если не «завершить для всех».
- **VPN off ОБЯЗАТЕЛЕН.** Смоук с VPN-on (немецкий exit eu1) дал `Post .../get_anonym_token: lookup login.vk.ru: i/o timeout` — чистый сетевой таймаут до VK, не капча. Резидентный РФ-IP = ключ (DC/VPN не проходят, урок 06-23).
- **Конфликт live-агент vs VPN-off:** машина Никиты ходит к Claude через наш VPN → live-сессия и VPN-off несовместимы. Решение: агент готовит one-click `.bat` (`VK-TURN-TEST.bat` на рабочем столе), «выходит»; Никита гонит offline (VPN off → решает капчу на localhost:8765 → VPN on → «done»); агент читает `vkturn_test.log`.
