# Spec 07: резервный доступ с мобильной сети (VLESS + REALITY)

## Проблема

- AmneziaWG / WireGuard по **UDP** и часть прокси на **мобильном интернете** (LTE/5G) не устанавливают соединение, при этом на Wi‑Fi те же конфиги работают.
- Гипотеза: на тракте мобильного оператора агрессивнее фильтруется **UDP** и/или распознаётся сигнатура протокола; нужен **TCP** с маскировкой под обычный TLS (HTTPS).

## Цель

- **Не ломать** существующие сценарии (main WireGuard, eu1 AmneziaWG на ПК/Wi‑Fi).
- Добавить **параллельный** транспорт: **VLESS + XTLS Vision + REALITY** (Xray-core) на eu1, порт **TCP** (предпочтительно 443, иначе 8443).
- Выдача пользователю через бота: команда **`/mobile_vpn`** — краткая инструкция + **сырая** ссылка `vless://` (вторым сообщением без HTML, чтобы не портить `&`).

## Не в scope

- Автоматическая выдача **отдельного UUID на каждого пользователя** (позже: скрипт/API к Xray). Первый этап — **одна общая ссылка** в `env_vars.txt` для доверенного круга; ротация — смена UUID на сервере + обновление переменной.
- Остановка или замена AmneziaWG.

## Требования

1. Перед установкой Xray на eu1 — **бэкап** AmneziaWG и текущего состояния (см. `docs/xray-vless-reality-eu1-deploy.md`, `docs/backup-restore.md`).
2. Inbound только **VLESS** + **reality** + **flow** `xtls-rprx-vision`; outbound `freedom`.
3. `dest` / `serverNames` — реальный сайт с валидным TLS на выбранном порту (в инструкции — пример; оператор подставляет свой после проверки).
4. Секреты не в Git: только `VLESS_REALITY_SHARE_URL` на сервере в `env_vars.txt`.

## Риски и митигация

| Риск | Митигация |
|------|-----------|
| Порт 443 занят | Использовать 8443; в ссылке указать тот же порт. |
| Блокировка IP eu1 по TCP | Запасной VPS/домен (вне спека; операционно). |
| Утечка общего UUID | Ротация UUID; позже — per-user в Xray API. |
| Неверный `dest` | Проверка `openssl s_client` с eu1 до выбора `dest`. |

## Чеклист проверки

- [ ] Бэкап AmneziaWG и `docker inspect` выполнен.
- [ ] `systemctl status xray` — active; `ss -tlnp` — слушает выбранный TCP-порт.
- [ ] С Wi‑Fi: клиент подключается, `ifconfig.me` показывает IP eu1.
- [ ] С мобильного интернета: то же самое.
- [ ] В боте: зарегистрированный пользователь получает `/mobile_vpn`, ссылка копируется целиком.
- [ ] ПК-пользователи продолжают использовать AmneziaWG без изменений.
