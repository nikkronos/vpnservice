# Альтернативные решения для Telegram-прокси (без гарантий)

**Цель этого файла** — зафиксировать найденные в интернете и чатах альтернативные способы ускорить/разблокировать Telegram, чтобы не держать всю информацию в голове.

**Важно:**

- Эти решения **не входят в официальный стек проекта VPN/Proxy**.
- Их **надёжность и работоспособность не гарантируются**.
- Использовать **только на свой риск**, с учётом безопасности аккаунта Telegram и трафика.
- Основной и рекомендованный способ в проекте — **Telegram через VPN (AmneziaWG)**, описанный в `docs/blocking-bypass-strategy.md` и `docs/step-by-step-plan-bypass.md`.

---

## 1. Локальный SOCKS5-прокси TG WS Proxy

- **Что это:** локальный SOCKS5‑прокси для Telegram Desktop, который подключается к `kws*.web.telegram.org` по WebSocket и помогает частично ускорить загрузку Telegram.
- **Репозиторий:** [`Flowseal/tg-ws-proxy`](https://github.com/Flowseal/tg-ws-proxy)
- **Идея работы (кратко):**
  - Локальный SOCKS5 поднимается на `127.0.0.1:1080`.
  - Telegram Desktop настраивается на этот SOCKS5.
  - Трафик к датацентрам Telegram уходит по WSS (WebSocket поверх TLS) через домены `kwsN.web.telegram.org`.
- **Плюсы:**
  - Open source, можно собрать из исходников.
  - Не требует покупки внешнего прокси/сервера.
- **Минусы и риски:**
  - Фокус на Telegram Desktop; для мобильных клиентов не всегда подходит.
  - Нужно доверять коду/сборке, следить за обновлениями.
  - Нет централизованного мониторинга, всё крутится локально у пользователя.
- **Статус для проекта:**
  - Найден как потенциальная альтернатива.
  - **Не интегрирован** в текущую архитектуру VPN/бота.
  - **Не проходил полноценное тестирование владельцем**; используется только как справочная опция.

## 2. Платные SOCKS5-прокси для Telegram (пример: shopproxy)

- **Что это:** сторонние платные прокси‑сервисы, которые продают IPv4‑адреса и авторизованный SOCKS5‑доступ, в т.ч. под Telegram.
- **Пример из чата:** `https://ru.shopproxy.net/buy-proxy/telegram/`  
  Информация ниже основана на сообщении из чата (опыт другого человека), **владелец проекта лично не проверял именно этот сервис**.

### 2.1. Сценарий использования (по описанию из чата)

- Вариант, который упоминали:
  - **Страна:** США.
  - **Протокол:** SOCKS5.
  - **Срок:** 30 дней.
  - Стоимость порядка ~72₽ в месяц.
  - Оплата через Tpay или QR‑код, при покупке указывается e‑mail (привязка к личному кабинету).

- После покупки в личном кабинете в разделе «Мои прокси» отображаются:
  - IP‑адрес.
  - Порт.
  - Логин и пароль.

### 2.2. Настройка в Telegram (по инструкции из сообщения)

**С компьютера (Telegram Desktop):**

1. Открыть **Настройки → Продвинутые настройки → Тип соединения → Использовать собственный прокси**.
2. Добавить прокси:
   - Тип: **SOCKS5**.
   - Сервер: значение `ip` из личного кабинета, например `123.123.123.123` (без двоеточия).
   - Порт: число после двоеточия в строке прокси, последние 4–5 цифр (например `12345`, без двоеточия).
   - Логин/пароль: те, что выдал провайдер.

**С телефона (мобильное приложение Telegram):**

1. **Настройки → Данные и память → Настройки прокси**.
2. Нажать **«Добавить прокси»** и выбрать **SOCKS5**.
3. Заполнить:
   - Сервер: IP из личного кабинета.
   - Порт: порт из строки прокси.
   - Логин и пароль: как указано в кабинете.

### 2.3. Замечания и риски

- Это **полностью сторонний сервис**:
  - Владелец прокси видит IP‑адреса, к которым идёт трафик.
  - Теоретически возможен перехват/логирование метаданных.
- **Риски для аккаунта Telegram:**
  - При агрессивном использовании или при попадании прокси в чёрные списки возможны дополнительные проверки от Telegram.
- **Статус для проекта:**
  - Зафиксирован как **альтернативный внешний вариант** на случай, если ничего больше не работает и нет времени поднимать свой сервер.
  - **Не рекомендуется как основной путь** в рамках проекта VPN/Proxy.
  - Детальная оценка надёжности и юрисдикционных рисков не проводилась.

## 3. Как относиться к этим вариантам в рамках проекта

- Официальная и поддерживаемая линия проекта:
  - **Telegram через VPN/AmneziaWG** (см. `docs/blocking-bypass-strategy.md`, `docs/step-by-step-plan-bypass.md`).
  - Максимальная автоматизация через Telegram‑бота и свои VPS.
- Эти альтернативы:
  - **Не меняют архитектуру проекта**.
  - Используются только как «памятка, что ещё можно попробовать», если РКН будет ежедневно ломать привычные решения.
  - При появлении нового опыта (успешные/неуспешные тесты) сюда можно добавлять краткие заметки и статусы: «проверено, работает/не работает».
