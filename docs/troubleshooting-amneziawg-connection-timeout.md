# Ошибка «Не удалось установить соединение» (тайм-аут) в AmneziaWG

## Симптомы

- На телефоне (AmneziaWG) при включении профиля: «Ожидание рукопожатия» → «Не удалось установить соединение. Попытка подключения завершилась по тайм-ауту через 12 секунд».
- На ПК (AmneziaVPN к тому же серверу 185.21.8.91) VPN может при этом работать нормально.
- В мониторинге по пользователю: 0 Б принято / 0 Б отправлено.

## Решение для пользователя

Команда **/regen** в боте — бот выдаёт новый конфиг. Старый конфиг в приложении удалить и заменить новым (импорт по инструкции для AmneziaWG).

Альтернатива (если есть доступ к рабочему ПК с AmneziaVPN): экспорт рабочего .conf через «Поделиться» и импорт на телефон (Файлы → долгое нажатие на .conf → Поделиться → AmneziaWG).

## Почему /regen исправляет ситуацию

- **/regen** на стороне бота: удаляется старый peer на сервере (eu1, AmneziaWG в Docker), создаётся новый peer с новыми ключами и параметрами обфускации, пользователю выдаётся новый .conf.
- Старый конфиг на телефоне перестал соответствовать серверу: либо ключи/параметры обфускации разъехались (например, после изменений на сервере или контейнере), либо запись peer на сервере была в «битом» состоянии (нет handshake). Новый peer и новый .conf восстанавливают полное совпадение клиент ↔ сервер, рукопожатие проходит.

## Почему на ПК проблемы не было

- ПК часто настроен через **AmneziaVPN с доступом по SSH** к серверу: приложение само управляет протоколом AmneziaWG на сервере и получает актуальное состояние. Конфиг по сути «живой», не разовая выдача из бота.
- Либо на ПК используется другой peer (другой IP в 10.8.1.x), который на сервере остался в порядке; «сломанным» был только один peer (например, 10.8.1.2), соответствующий конфигу, выданному ботом для телефона.
- В итоге один и тот же сервер (порт 39580, контейнер amnezia-awg2) обслуживает и ПК, и телефон; проблема была не в сервере, а в несовпадении конфига на телефоне с записью peer на сервере.

## Рассылка уведомления пользователям

Владелец может отправить всем пользователям уведомление о проблеме и решении через команду бота **/broadcast**. Текст сообщения зашит в боте (уведомление о проблеме VPN и рекомендация выполнить /regen).
