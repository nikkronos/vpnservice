# ═══ §1 · ЧТО ЭТО И ЗАЧЕМ

**Проблема.** Жёсткий мобильный whitelist (БС — у тест-контакта Ани, Муром) пропускает трафик
ТОЛЬКО к whitelisted-сервисам (Госуслуги, VK, и т.п.) и **режет сам прокачиваемый трафик** всего
остального — «пинг есть, туннель не несёт». Бьёт всё: статика igareck-150 провалена, наши
VLESS-REALITY и AmneziaWG срезаны (датацентр-IP не в белом списке). См. `project_vpn_rkn_ip_block`,
`project_vpn_hard_whitelist_unbeaten`.

**Идея vk-turn** (`cacggghp/vk-turn-proxy` «Good TURN»). VK-звонки — whitelisted (VK в белом списке).
vk-turn гонит WireGuard-трафик **внутри настоящего VK-звонка**: пакеты шифруются DTLS и идут через
**VK TURN-релей** (STUN ChannelData) на наш сервер. На проводе у клиента видно только «VK-звонок» —
WG там нет. Поэтому **блок WireGuard в РФ не помеха** (WG едет по loopback). Если whitelist пропускает
реальный VK-звонок с данными — он несёт и наш туннель.

**Топология (полная цепочка):**
```
[WG-клиент] → 127.0.0.1:9000 (vk-turn CLIENT, loopback)
           → DTLS/STUN → [VK TURN-релей, whitelisted]
           → UDP → [наш eu1 :56000, vk-turn SERVER]
           → 127.0.0.1:<wg_port> (наш WG-сервер) → NAT → интернет
```
Клиент **анонимно** входит в VK-звонок по ссылке (логин VK не нужен) и получает TURN-креды.
Клиент НЕ коннектится к серверу напрямую — только через VK-релей → поэтому whitelist его не видит.

# ═══ §4 · ЧТО НЕ ДОКАЗАНО И ПОЧЕМУ

**Главный вопрос — пробивает ли vk-turn ЖЁСТКИЙ whitelist — остался ОТКРЫТЫМ.** Тоннель доказан на
открытой сети; пропустит ли его конкретный БС (не срежет ли релей-данные, как срезал всё остальное) —
не проверено.

**Почему сорвалось у Ани (топологический тупик):** её жёсткий whitelist — только на МОБИЛЕ (айфон).
Ноут — в нормальной сети (всё открывается). Тетеринг ноута к айфону невозможен (нет сигнала там, где
ноут). → единственное БС-устройство = **iPhone**, а vk-turn на iOS = **iSH/sideload** (нереалистично
для нетехнического; рабочий-с-капчей клиент только CLI; iOS-GUI-клиенты вероятно сломаны новой капчей).

**Почему нельзя «без реального БС»:** симулировать жёсткий whitelist с открытой сети нельзя.
Самодельная заглушка (просто блок IP) НЕ воспроизведёт реальную DPI-фильтрацию релей-трафика → даст
ложную уверенность (пройдёт у нас, ляжет на настоящем БС). **Валидный тест — только на реальном
БС-устройстве** (нормальном для инструментовки: Windows/Android; iOS = тупик).

# ═══ §5 · КАК ТЕСТИТЬ НА РЕАЛЬНОМ БС (runbook)

**Кто:** владелец сам, когда БС введут на его сеть (под жёстким БС его Wi-Fi+VPN к Claude сохраняется →
координация работает). ИЛИ любой Android-юзер на реальном жёстком БС.

**Шаги (всё готово заранее, см. §8):**
1. Пинг агенту: «БС пришёл, подними vk-turn» → агент за ~1 мин поднимает сервер на eu1 (`up.sh`).
2. Создать СВЕЖИЙ VK-звонок (VK → Звонки → Создать звонок) и **остаться в нём** (пустой VK закрывает →
   `Call not found`; см. §7). Дать ссылку.
3. Выключить обычный VPN. Запустить `VK-TURN-VALIDATE.bat` → вставить ссылку (правый клик) → Enter →
   решить капчу (вкладка откроется сама; если вторая — решить и её). Окно НЕ закрывать.
4. ~20с → в WireGuard включить туннель `nikita-vkt`.
5. Открыть сайт, что без туннеля не грузился. **Грузится = vk-turn ПРОБИЛ жёсткий БС.** Не грузится =
   не пробил («пинг есть, туннель не несёт»).
6. Агент подтверждает со стороны eu1 (`wg show wg-vkt` handshake+трафик пира + `server.log`).
7. После: выключить туннель, закрыть окно → агент сворачивает сервер (`down.sh`).

Локальная копия рунбука: `C:\Users\krono\vkturn\RUNBOOK-when-BS-hits.txt`.

# ═══ §6 · ТЕХ-ДЕТАЛИ (конфиги / капча)

**Бинари:** prebuilt релиз форка **`NikKuz99/vk-turn-proxy` v1.8.4** (Go; client/server на все платформы).
Версия-баннер `captcha-fix=v2 tls-ca-fix=v1`.

**Фикс капчи (PR #183).** VK сменил формат на шаге `calls.getAnonymousToken`: ошибка несёт `redirect_uri`,
но НЕ несёт `captcha_sid`/`captcha_img`. Старый парсер падал `missing captcha_sid`, не доходя до решателя.
v1.8.4: `ParseVkCaptchaError` делает поля опциональными + ручной решатель (`-manual-captcha`, отдаёт
капчу на `http://localhost:8765`) + Playwright-авто-решатель (`-captcha-solver`, реальный Chromium —
нужен из-за FingerprintJS) + embedded HARICA CA-бандл (Android TLS).

**Сервер (eu1, изолированно):**
```
wg-quick: /etc/wireguard/wg-vkt.conf — iface wg-vkt, 10.99.99.0/24, ListenPort 51899,
          PostUp/PostDown: MASQUERADE 10.99.99.0/24 → eth0 + FORWARD ACCEPT
vk-turn:  server -listen 0.0.0.0:56000 -connect 127.0.0.1:51899
пиры:     10.99.99.2 (владелец), 10.99.99.3 (Аня/резерв)
inbound:  56000/udp (приходит от VK-релея; ufw на eu1 нет)
```

**Клиентский WG-конфиг** (`nikita-vkt.conf`, импортирован в WireGuard for Windows):
```
[Interface] Address 10.99.99.2/32; DNS 1.1.1.1; MTU 1280
[Peer]      Endpoint = 127.0.0.1:9000   (локальный vk-turn CLIENT, НЕ наш сервер)
            AllowedIPs = 772-CIDR VK-исключение (из README репо)
            PersistentKeepalive = 25
```
**Важно про AllowedIPs:** туннелим ВСЁ, КРОМЕ диапазонов VK — чтобы трафик клиента к VK-релею шёл
DIRECT, иначе цикл (WG ловил бы собственные пакеты к релею). Проверено: VK-релей-диапазон
`90.156.232.0/21` — gap (вне AllowedIPs → direct). Брать ПОЛНУЮ (длинную) строку из README, не короткую.

**Запуск клиента (что внутри `.bat`):**
```
client-windows-amd64.exe -vk-link "<живая VK-ссылка>" -manual-captcha -debug \
  -peer 185.21.8.91:56000 -listen 127.0.0.1:9000
```

# ═══ §7 · ОГРАНИЧЕНИЯ / ПОТОЛОК

Даже при зелёном пробое жёсткого БС — это **личный/боутик-хак, НЕ масс-продукт** (обход анти-масштабен,
вывод 06-25, 3 подтверждения; `project_vpn_hard_whitelist_unbeaten`). Операционные барьеры:
- **Креды VK ~9 мин** (`expires in 8m59s`) → переавторизация, вероятно повторная капча → неприсматриваемо
  непригодно. Авто-solver = Playwright (ловит `status:BOT` с датацентр-IP → нужен резид. IP + браузер).
- **Нужен живой VK-звонок** на момент теста; пустой звонок VK закрывает → `error_code:9000 Call not found`
  (на этом споткнулись 06-30, ложная тревога). Для продукта — «живой звонок 24/7».
- **iOS почти не инструментуется** (iSH/sideload); Android проще. Happ выпилен из РФ App Store — iOS в РФ
  структурно хрупок.
- Итог: коммерческая форма максимум — бутик (пара хард-БС юзеров вручную, премиум, нянчить). Доход не
  масштабирует. Поэтому трек запаркован по СТРАТЕГИИ, не по технике.
