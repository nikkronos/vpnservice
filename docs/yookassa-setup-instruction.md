# ЮKassa — пошаговая настройка для самозанятого

> Документ для владельца. Цель: подключить ЮKassa как платёжного провайдера и получить `Shop ID` + `Secret key` для интеграции в код (Phase 4).
>
> Имеешь самозанятость (по `kronos-lolly@yandex.ru`) — этот путь самый простой и быстрый.

---

## Шаг 0. Предусловие — «Мой налог»

Убедись, что **самозанятость активна** в приложении/сайте «Мой налог» (lknpd.nalog.ru). Если ещё не оформлено — оформи (5 минут, паспорт + ИНН). Без этого ЮKassa не сможет автоматически выставлять чеки от твоего имени.

---

## Шаг 1. Регистрация на ЮKassa

1. Зайди на **yookassa.ru** → «Подключиться».
2. Выбери форму бизнеса: **«Самозанятый» (НПД)**.
3. Подтверди номер телефона + email (используй `kronos-lolly@yandex.ru` или другой рабочий).
4. Заполни персональные данные:
   - ФИО, ИНН (как в «Мой налог»).
   - Реквизиты для получения денег: **карта** (обычная дебетовка) или расчётный счёт. Самозанятому удобнее карта.
5. Подпиши оферту ЮKassa (электронно).
6. **Дай согласие на интеграцию с ФНС** — это разрешит ЮKassa формировать чеки от твоего имени в «Мой налог» автоматически после каждого платежа (обязательное требование закона по самозанятости).

ЮKassa проверит данные (обычно несколько часов–1 день).

---

## Шаг 2. Создание магазина

После одобрения:

1. Личный кабинет ЮKassa → **«Магазины» → «Создать магазин»**.
2. Заполни:
   - **Название**: `VPN Kronos` (или `Kronos Subscriptions`, нейтрально).
   - **URL сайта**: `https://supportkronos.online:8443` (наш ЛК).
   - **Описание услуги**: ⚠️ **не пиши явно «VPN»** — ЮKassa теоретически может задать вопросы. Лучше нейтральные формулировки:
     - `Доступ к интернет-сервису по подписке`
     - `Цифровой сервис подписочного типа`
     - `Сервис для разблокировки сетевых ограничений`
   - **Категория**: «Цифровые услуги / IT / SaaS» (выбери ближайшую).
3. Подтверди создание.

---

## Шаг 3. Получение API-ключей

1. В личном кабинете магазина → **«Настройки» → «Ключи API»** (или «Интеграция → API»).
2. Найди / создай **боевой Secret key**. ЮKassa покажет его **один раз** — сразу сохрани.
3. **Shop ID** виден в настройках магазина (числовой идентификатор).

**Запиши:**
- `YOOKASSA_SHOP_ID` = ...
- `YOOKASSA_SECRET_KEY` = `live_...` (начинается с `live_` для боевого режима)

⚠️ **Secret key — секрет уровня пароля.** Не публикуй, не коммить, не показывай скриншотами. Пришли в личку, когда получишь.

---

## Шаг 4. Настройка вебхука

ЮKassa уведомляет наш сервер о статусе платежа через HTTP-вебхук.

1. В настройках магазина → **«HTTP-уведомления»** (или «Webhook»).
2. URL для уведомлений: **`https://supportkronos.online:8443/api/billing/yookassa-webhook`**
3. Подписаться на события (галочки): минимум — `payment.succeeded`, `payment.canceled`. Полезно также `refund.succeeded` (на будущее).
4. Сохранить.

ЮKassa будет POST'ить JSON на этот URL. Мы реализуем обработчик (Phase 3e).

---

## Шаг 5. Тестовый магазин (опционально, рекомендуется)

В ЮKassa есть отдельный «тестовый магазин» — параллельно боевому. Свой `Shop ID` + `Secret key` (начинается с `test_`). Платежи проходят через тестовые карты ЮKassa без списания денег.

- Рекомендую: сначала прислать **тестовый** Shop ID + Secret. Протестируем полный цикл (создание платежа → ЮKassa-страница → webhook → продление). Когда работает — переключим на боевой.

## Что я делаю на своей стороне (Phase 3e)

После получения ключей:
1. Добавлю в `env_vars.txt` на сервере (gitignored).
2. Реализую `POST /api/billing/create-payment` — создаёт платёж через ЮKassa API, возвращает `confirmation_url` (страница оплаты ЮKassa).
3. Реализую `POST /api/billing/yookassa-webhook` — обрабатывает `payment.succeeded`, продлевает `expires_at`, записывает в `payments`, начисляет реферал-бонус если применимо.
4. Кнопка «Продлить на 30 дней — 200 ₽» в ЛК — рядом с подпиской.
5. Тест в режиме `test_` (тестовые карты), потом переключение на `live_`.

## Возможные проблемы (на что обращать внимание)

- **Модерация ЮKassa может задать вопросы** по типу услуги. Если попросят детальнее — отвечай: «онлайн-сервис для обхода ограничений доступа к сайтам, по подписке». Слова «VPN» избегать прямо, но если спросят — не лгать (это не запрещено, просто привлекает внимание).
- **Чеки**: ЮKassa формирует автоматически после каждого платежа через интеграцию с «Мой налог». Налог 4% (физлица) или 6% (юрлица) — самозанятый платит из дохода. Проверяй чеки в «Мой налог».
- **Лимит самозанятого**: 2.4 млн ₽/год. При выходе за лимит — нужно переходить на ИП.
- **Возвраты**: ЮKassa API позволяет вернуть деньги. Мы это пока не делаем, но Можем добавить кнопку «отменить подписку» в ЛК (без авто-возврата за неиспользованные дни — простая модель).

- **Tinkoff Pay / Sber Pay** — если ИП в будущем.
- **Stars (Telegram)** — у нас уже в плане как 2-я кнопка (Phase 3g).
- **Крипта (USDT)** — Cryptomus / CryptoCloud — для анонимных.
